## v2.9.0 (2026-03-11) - Historical Sensor Aggregation
Priority: medium

Sensor data sent to Firebase is now pre-aggregated per UTC hour, dramatically
reducing database read costs and enabling efficient long-range charting in the
Admin Dashboard.

### New
- Every incoming sensor reading now overwrites a lightweight `live_readings` path
  in Firebase so dashboards always see the latest values without querying
  the raw history
- Completed UTC-hour buckets are automatically written to a new
  `historical_aggregates` path containing average, minimum, and maximum values
  for temperature, humidity, and CO₂ per room
- Partial buckets are safely flushed to Firebase on device shutdown so no data
  is lost during reboots

### Improved
- Firebase bandwidth usage is significantly lower — one write per hour per room
  instead of one write every 5 seconds
- Admin Dashboard charts now load weeks or months of data without hitting
  Firebase read limits

### Known Limitations
- Historical aggregates only accumulate from the moment this firmware version
  is first run; older raw data must be migrated separately using the
  `migrate_to_historical_aggregates.py` script
- The legacy `sensor_data/{serial}/{room}/{timestamp}` writes are still enabled
  for backward compatibility and can be disabled via user preferences
