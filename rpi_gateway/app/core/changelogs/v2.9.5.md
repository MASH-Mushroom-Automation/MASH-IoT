## v2.9.5 (2026-03-22) - Firebase Queue Command Control
Priority: high

Added Firebase command queue polling in the gateway so web fallback commands now reach the actuators when MQTT is unavailable.

### Fixed
- Web-based actuator toggles now get processed by the gateway instead of staying queued in Firebase.
- MQTT and Firebase remote commands now share the same actuator execution path.
- Processed cloud commands are removed from the queue after successful delivery.
