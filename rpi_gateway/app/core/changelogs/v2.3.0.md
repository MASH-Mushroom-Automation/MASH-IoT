## v2.3.0 (2026-02-11) - Version Management
Priority: medium

Refactored version management system for scalable changelog handling and future OTA update support.

### New
- Per-version changelog files stored in individual files for on-demand loading
- Changelog API now supports query parameters for efficient data retrieval
  - Fetch only the current version changelog (default)
  - Fetch changelogs since a specific version (for "what's new" popups)
  - Fetch a specific version's changelog
  - Fetch the last N versions
- Priority levels on each release (high, medium, low) for future update notifications
- Consistent changelog format with standardized tags (New, Fixed, Improved, Changed)

### Changed
- Changelogs are now stored as separate files instead of one large text block
- Changelog API response is now structured JSON with per-version entries
- Version info display cleaned up (removed special characters from output)
- Advanced recovery feature flag updated to reflect I2C recovery implementation

### Improved
- Reduced memory usage by loading changelog content on demand
- Cleaner version module with compact registry metadata
- User-friendly changelog language (no technical function names)
