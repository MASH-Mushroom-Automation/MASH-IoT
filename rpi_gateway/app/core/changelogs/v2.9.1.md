## v2.9.1 (2026-03-11) - Remove Redundant Live Readings Path
Priority: low

Removed a duplicate real-time data path that was writing the same sensor
values to two different locations in the cloud database simultaneously.

### Changed
- The aggregator no longer writes to a separate per-room live readings path;
  the existing device-level latest reading (already used by all apps) is the
  sole real-time data source

### Improved
- Reduced Firebase Realtime Database write operations by ~50% per sensor
  reading cycle (was 3 writes: raw data + latest_reading + live_readings;
  now 2 writes: raw data + latest_reading)
- Simplified aggregator internals — no more live-reading side-effect on every
  accumulation call
